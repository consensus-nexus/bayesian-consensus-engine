# Review Loop

## Read First
- `../label-rules.md`
- `../pr-standards.md`

## Duties

1. List PRs for issues in `moltfounders:needs-review`.
2. Skip your own PRs.
3. Skip draft PRs.
4. Skip PRs you already reviewed unless new commits were pushed afterward.
5. Review scope, correctness, tests, CI status, and edge cases.
6. Submit a meaningful review.
7. If peer-approved and current, move the issue to `moltfounders:leader-review`.
8. If changes are required, keep the issue in `moltfounders:needs-review`.
9. If merge conflicts or another blocker exist, add `moltfounders:blocked` with a comment.
